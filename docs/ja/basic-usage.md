# 基本的な使い方

## メッセージを送信する

AntiCrow の基本的な使い方は、Discord のテキストチャンネルにメッセージを送信するだけです。

### 手順

1. Discord で AntiCrow Bot がいるサーバーを開く
2. テキストチャンネルにタスク内容を自然言語で投稿
3. AI が実行計画を生成し、確認メッセージを表示
4. ✅ （承認）または ❌ （却下）で実行を制御
5. 実行完了後、結果が Discord に通知される

### 例

```
READMEを日本語と英語で作成してください
```

```
src/utils.ts のバグを修正してください。null チェックが漏れています
```

```
テストを全部実行して、失敗したものを修正してください
```

## 確認フロー

メッセージを送信すると、AntiCrow は以下のフローで処理します:

1. **📥 メッセージ受信** — メッセージを解析
2. **🧠 計画生成** — AI が実行計画を JSON で生成
3. **📋 確認表示** — 実行内容の確認メッセージを Discord に表示
4. **✅ 承認 / ❌ 却下** — ユーザーが判断
5. **⏳ 実行** — Antigravity がタスクを実行
6. **📊 進捗通知** — リアルタイムで進捗を表示
7. **✅ 完了通知** — 結果を Discord に通知

## 承認ボタン

| ボタン | 動作 |
|--------|------|
| ✅ 承認 | タスクを実行する |
| ❌ 却下 | タスクをキャンセルする |
| 🤖 エージェントに任せる | AI の判断で実行内容を決定 |

## ファイルの添付

Discord メッセージに画像やテキストファイルを添付して指示できます。AI がファイルの内容を読み取り、タスクに反映します。

```
添付画像のデザインを参考に、ランディングページを作成してください
```

## キューイング

処理中に追加のメッセージを送信すると、自動的にキューに追加されます。前のタスクが完了すると、キュー内の次のタスクが順番に処理されます。

キューの状態は `/queue` コマンドで確認できます。

## ワークスペースの管理

AntiCrow はワークスペースごとに Discord カテゴリを自動作成します。各カテゴリ内に `#agent-chat` チャンネルが作られ、そこでやり取りします。

- `/workspace` コマンドでワークスペース一覧を確認
- 新規ワークスペースは Discord から直接作成可能
- 一定期間使用されていないカテゴリは自動アーカイブ（設定可能）

## タスクの停止

実行中のタスクを中断したい場合は、`/stop` コマンドを使います:

```
/stop
```

- 実行中のジョブ → キャンセル
- キュー内の待機ジョブ → 保持

## コツ

> 💡 **1メッセージ = 1タスク** で送信すると精度が上がります

> 📎 画像やテキストファイルを添付して指示できます

> ⏱️ 処理中に追加メッセージを送ると自動でキューに追加されます

> ⏹️ タスクをやめたい時は `/stop` を使ってください
