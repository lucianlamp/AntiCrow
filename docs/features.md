# AntiCrow 機能ドキュメント

> 最終更新: 2026-02-20

## 概要

AntiCrow は、Discord から AI コーディングアシスタント **Antigravity** を操作するブリッジ拡張機能です。

Discord のチャンネルに自然言語でメッセージを送るだけで、Antigravity にタスクを実行させ、結果を Discord に返します。即時実行だけでなく、cron 式による定期実行やスケジュール管理にも対応しています。

### 主な特徴

- **自然言語でタスク指示** — 技術的なコマンドを覚える必要なし
- **実行計画の自動生成** — AI がメッセージを解析し、実行計画を自動生成
- **承認フロー** — 実行前にユーザーの確認を挟める安全設計
- **複数ワークスペース対応** — プロジェクトごとに独立したチャンネルで管理
- **リアルタイム進捗通知** — 処理の進行状況を Discord にリアルタイムで表示
- **ファイル添付対応** — 画像やテキストファイルを添付してコンテキストを共有

## 主要機能

### 1. 自然言語タスク実行

Discord チャンネルにメッセージを送信すると、AntiCrow が以下のフローで処理します:

1. **メッセージ受信** — Discord チャンネルのメッセージを受信
2. **実行計画生成** — AI がメッセージを解析し、実行計画（Plan）を自動生成
3. **確認（オプション）** — 実行計画をユーザーに提示し、承認/却下を待つ
4. **実行** — Antigravity にプロンプトを送信し、タスクを実行
5. **結果通知** — 実行結果を Discord に送信

**添付ファイル対応**: 画像（PNG, JPG, GIF, WebP）やテキストファイル（.txt, .md, .json, .ts, .py など）をメッセージに添付することで、AI にコンテキストを共有できます。

### 2. 実行計画と承認フロー

タスクの実行前に、AI が自動生成した実行計画をユーザーに確認してもらうことができます。

- **即時実行**: `cron: "now"` — すぐに実行
- **定期実行**: `cron: "0 9 * * 1-5"` — cron 式でスケジュール登録
- **承認モード**: ✅ 承認 / ❌ 却下のリアクションで操作
- **選択モード**: 複数の選択肢から選んで実行（single / multi / all）

### 3. スラッシュコマンド一覧

| コマンド | 説明 |
|---|---|
| `/status` | Bot・接続・キュー状態を一覧表示 |
| `/schedules` | 定期実行の一覧・管理パネルを表示 |
| `/stop` | 実行中のタスクを停止 |
| `/newchat` | Antigravity で新しいチャットセッションを開始 |
| `/workspace` | 検出された Antigravity ワークスペース一覧を表示 |
| `/queue` | メッセージ処理キュー・実行キューの詳細を表示 |
| `/template` | テンプレート一覧を表示・管理 |
| `/model` | 利用可能な AI モデル一覧を表示・切り替え |
| `/mode` | AI モード切替（Planning / Fast） |
| `/history` | Antigravity の会話履歴を表示・切り替え |
| `/suggest` | プロジェクトを分析して次にやることを提案 |
| `/help` | AntiCrow のコマンド一覧と使い方を表示 |
| `/pro` | Pro ライセンスの管理・購入・キー入力 |
| `/screenshot` | 現在の画面のスクリーンショットを取得 |
| `/soul` | SOUL.md（カスタマイズ設定）を編集 |
| `/team` | エージェントチームモードの管理 |

### 4. テンプレート機能

よく使うタスクをテンプレートとして保存し、ワンクリックで再利用できます。

- **テンプレート保存** — 任意のプロンプトをテンプレートとして登録
- **テンプレート実行** — ボタン操作で保存済みテンプレートを即座に実行
- **テンプレート編集** — モーダルダイアログでテンプレートの内容を編集
- **テンプレート削除** — 不要になったテンプレートを削除
- `/template` コマンドでインタラクティブな管理パネルを表示

### 5. スケジュール実行

cron 式を使った定期実行に対応しています。

- **cron 式** — 標準5項目形式（分 時 日 月 曜日）
- **タイムゾーン対応** — 設定した IANA タイムゾーンで実行（デフォルト: `Asia/Tokyo`）
- **スケジュール管理** — `/schedules` コマンドで登録・解除・一覧
- **永続化** — 拡張機能の再起動時にスケジュールを自動復元

### 6. モデル・モード切替

Discord から AI モデルやモードをリアルタイムに切り替えられます。

- **モデル切替** (`/model`) — 利用可能なモデルの一覧をボタン付きで表示。ワンクリックで切り替え
- **モード切替** (`/mode`) — Planning / Fast モードをボタンで切り替え
- **クォータ表示** — 各モデルの使用量/上限をリアルタイム表示

### 7. 会話履歴管理

Antigravity の会話履歴を Discord から操作できます。

- **履歴一覧** (`/history`) — 過去の会話をページネーション付きで表示
- **会話切り替え** — ボタン操作で任意の過去の会話に切り替え
- **新規チャット** (`/newchat`) — 新しいチャットセッションを開始

### 8. カスタマイズ機能

AntiCrow の応答スタイルをカスタマイズできます。

- **口調変更** — 「ずんだもんの口調にして」などの自然言語で変更可能
- **呼び方設定** — ユーザーの呼び方を変更
- **設定ファイル** — `~/.anticrow/SOUL.md` でカスタマイズ設定を管理
- **自動バックアップ** — カスタマイズ変更時に自動でバックアップを作成

### 9. メモリー機能（学習・記憶の永続化）

AI エージェントが過去のタスクから学んだ教訓や重要な発見を **自動的に記憶** し、次回以降のタスクに活かします。

#### メモリーの種類

| 種類 | 保存場所 | 用途 |
|---|---|---|
| グローバル記憶 | `~/.anticrow/MEMORY.md` | 全プロジェクト共通の学び（ユーザーの好み、汎用パターン等） |
| ワークスペース記憶 | `{workspace}/.anticrow/MEMORY.md` | プロジェクト固有の学び（ビルド手順、バグ回避策等） |

#### 記録される内容

- 重要な技術的決定とその理由
- 繰り返し発生した問題の解決パターン
- プロジェクト固有のビルド手順や注意点
- 失敗した手法と代替策

#### SOUL.md との違い

| | SOUL.md | MEMORY.md |
|---|---|---|
| 目的 | AI の性格・口調・コーディングスタイルを指定 | 過去のタスクから学んだ教訓を蓄積 |
| 誰が書く？ | ユーザーが手動で記述 | AI が自動的に記録（ユーザーも編集可） |
| 内容の例 | 「日本語で回答して」「TypeScript を使って」 | 「このプロジェクトでは esbuild でバンドルする」 |

#### その他の特徴

- **自動記録** — タスク実行完了時に重要な知見を自動で記録
- **サイズ管理** — 上限（50KB）を超えた場合は古いエントリを自動アーカイブ
- **平文 Markdown** — いつでも確認・編集・削除可能

### 10. 複数ワークスペース対応

複数のプロジェクト（ワークスペース）を同時に管理できます。

- **自動検出** — Antigravity で開いているワークスペースを自動検出
- **ワークスペース別チャンネル** — プロジェクトごとに独立した Discord チャンネルを自動作成
- **ワークスペース切り替え** — `/workspace` コマンドでボタン操作により切り替え
- **パス自動学習** — ワークスペースのフォルダパスを自動的に学習・記憶

### 11. クォータ管理

AI モデルの使用量を確認できます。

- **使用状況表示** — 各モデルの使用量と上限を表示
- **アカウントレベル** — 現在のアカウントレベル（Free / Pro 等）を確認
- **モデル一覧連携** — `/model` コマンド内でクォータ情報を自動表示

### 12. カテゴリ自動アーカイブ

一定期間使用されていないワークスペースカテゴリーを自動的にクリーンアップします。

- **自動削除** — 設定日数（デフォルト: 7日）未使用のカテゴリーを自動削除
- **安全対策** — アクティブなスケジュールがあるカテゴリーは保護
- **全削除防止** — 全カテゴリーが対象になる場合は安全のため実行しない

### 13. アクセス制御

Discord ユーザー ID ベースのアクセス制御により、許可されたユーザーのみがタスクを実行できます。

- **ホワイトリスト方式** — 許可するユーザー ID を設定
- **デフォルト全拒否** — 未設定時は全ユーザーのアクセスを拒否（セキュリティ保護）

### 14. 自動操作

AntiCrow 経由のジョブ実行中に、Antigravity の確認ダイアログを自動的に処理します。

- **Continue / Allow の自動承認** — AI の提案を自動的に承認
- **ジョブ実行中のみ動作** — 手動操作時は自動承認されない
- **設定で有効化** — `antiCrow.autoOperation` で明示的に有効化が必要

### 15. リトライ機能

タイムアウトや接続断が発生した場合の自動リトライに対応しています。

- **自動リトライ** — 設定回数まで自動的に再実行
- **デフォルト無効** — `antiCrow.maxRetries: 0` でリトライ無効（安全設計）

### 16. ライセンス管理

Convex + Clerk + Stripe ベースのライセンス管理に対応しています。

- **ライセンス情報表示** — `AntiCrow: License Info` コマンドで確認
- **ログイン/ログアウト** — `AntiCrow: License Login` / `License Logout` コマンド
- **ライセンス購入** — `AntiCrow: Purchase License` コマンドで購入ページへ

## 設定項目

AntiCrow の動作は以下の設定項目でカスタマイズできます。

| 設定キー | 型 | デフォルト | 説明 |
|---|---|---|---|
| `antiCrow.botToken` | boolean | `false` | Bot Token 設定状態（SecretStorage に暗号化保存） |
| `antiCrow.timezone` | string | `""` | CRON 実行のタイムゾーン（IANA 形式）。空の場合は OS から自動取得 |
| `antiCrow.responseTimeoutMs` | number | `1800000` | アイドルタイムアウト（ミリ秒）。進捗報告がある限りリセット。デフォルト 30 分 |
| `antiCrow.autoStart` | boolean | `true` | 起動時に自動でブリッジを開始 |
| `antiCrow.clientId` | string | `""` | Discord Application の Client ID（スラッシュコマンド登録用） |
| `antiCrow.categoryArchiveDays` | number | `7` | ワークスペースカテゴリーの自動アーカイブ日数。0 で無効 |
| `antiCrow.allowedUserIds` | string[] | `[]` | メッセージ処理を許可する Discord ユーザー ID 一覧。空は全拒否 |
| `antiCrow.autoOperation` | boolean | `false` | 自動操作（Continue / Allow の自動承認）を有効にする |
| `antiCrow.maxRetries` | number | `0` | タイムアウト時の自動リトライ回数。0 でリトライ無効 |
| `antiCrow.convexUrl` | string | `""` | Convex デプロイメント URL（ライセンス管理用） |

## Discord チャンネル構成

AntiCrow はワークスペースごとに Discord チャンネルを自動作成・管理します。

### カテゴリー構造

```
🤖 my-project        ← ワークスペースカテゴリー（自動作成）
  └ #agent-chat      ← メインチャネル（タスク指示用）

🤖 another-project   ← 別のワークスペース
  └ #agent-chat

Schedules            ← スケジュール専用カテゴリー（必要に応じて自動作成）
  └ #daily-report    ← 定期実行タスクのチャンネル
```

### チャンネルの種類

- **#agent-chat** — ワークスペースのメインチャンネル。自然言語でタスクを指示
- **Plan チャンネル** — 個別タスクごとに自動作成されるチャンネル。実行計画の確認・結果の表示に使用

## 使い方のコツ

- 💡 **1 メッセージ = 1 タスク**で送信すると精度が上がります
- 📎 **画像やテキストファイルを添付**して指示できます
- ⏱️ **処理中に追加メッセージ**を送ると自動でキューに追加されます
- ⏹️ **タスクをやめたい時**は `/stop` を使ってください
- 🔄 **モデルを切り替えたい時**は `/model` でワンクリック切り替え
- 📋 **よく使うタスク**は `/template` からテンプレートとして保存できます
