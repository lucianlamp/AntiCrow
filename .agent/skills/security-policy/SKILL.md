# セキュリティポリシー: 内部実装の秘匿

AntiCrow は一般公開を前提とした拡張機能です。
内部の仕組みがユーザーに推測・追跡されないよう、以下のルールを**必ず**守ってください。

---

## 基本原則

- ユーザーが触れる場所（README、設定画面、エラーメッセージ、プロンプト）に**内部実装の詳細を露出しない**
- 仕組みをたどれるようなヒント（ファイルパス、プロトコル名、ポート番号）を一切含めない
- AI に送信するプロンプトにも内部のファイル構造やパスを含めない

---

## 禁止事項

### ユーザー向けテキスト（README、package.json の description、エラーメッセージ、UI）

以下の単語・表現を使用しないこと:

| ❌ 禁止 | ✅ 代替 |
|---|---|
| CDP / Chrome DevTools Protocol | （記載しない） |
| ポート番号 / port / 9222 / 9333 | （記載しない） |
| WebSocket | （記載しない） |
| `--remote-debugging-port` | （記載しない） |
| ファイルパス（VSIX 内部構造） | （記載しない） |
| `.anticrow/` ディレクトリ構造 | （記載しない） |
| `context.extensionPath` | （記載しない） |
| IPC / ファイルIPC | （記載しない） |
| アーキテクチャ図（内部ファイル一覧） | （記載しない） |


### 設定キー名（package.json の configuration）

- `cdp`、`port`、`websocket` などの内部実装を示す単語を設定キー名に使わない
- 例: `cdpResponseTimeoutMs` → `responseTimeoutMs`

---

## 許可されている記載

- 機能の説明（何ができるか）
- Discord Bot のセットアップ手順
- チャンネル構成
- 使い方
- カスタマイズ方法（`~/.anticrow/ANTICROW.md`）
- トラブルシューティング（ただし内部実装に踏み込まないこと）

---

## コード難読化（リバースエンジニアリング対策）

VSIX 配布時にソースコードが読み取られることを防ぐため、以下を**必ず**維持すること:

### esbuild 設定（`esbuild.js`）

| 設定 | 必須値 | 理由 |
|---|---|---|
| `minify` | `true` | コードを圧縮・短縮して可読性を下げる |
| `sourcemap` | `false` | デバッグ情報（元ソースとの対応）を排除 |

### `.vscodeignore`

以下のファイルを VSIX パッケージから**必ず除外**すること:

- `src/**` — TypeScript ソースコード
- `**/*.ts` — すべての TypeScript ファイル
- `tsconfig.json`, `esbuild.js` — ビルド設定
- `.agent/**` — AI エージェント設定（スキル・ワークフロー含む）
- `**/__tests__/**` — テストコード
- `*.map` — ソースマップファイル

### 禁止事項

- `minify: false` に戻さないこと（デバッグ時も VSIX ビルドは minify 有効のまま）
- `sourcemap: true` にしないこと（開発中のデバッグは console.log / OutputChannel で対応）
- `.vscodeignore` から `src/**` を削除しないこと

---

## チェックリスト（変更時に確認）

コード変更やドキュメント追加時に以下を確認してください:

- [ ] README.md に内部実装の詳細が含まれていないか
- [ ] package.json の設定 description に CDP/ポートなど内部用語がないか
- [ ] エラーメッセージにファイルパスやプロトコル名が含まれていないか
- [ ] ログメッセージは OutputChannel のみで、ユーザー向け UI には出さないか

---

## 背景

この拡張機能は Antigravity のエコシステム上で動作し、内部的に複数のプロトコルと通信方式を使用しています。
これらの実装詳細が公開されると、セキュリティリスクや模倣のリスクにつながります。
ユーザーにとっては「Discord で指示 → 結果が返る」というシンプルな体験だけが重要であり、
その裏側の仕組みは知る必要がありません。
